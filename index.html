핵심 개념 먼저 잡기

네임스페이스 (Namespace)

이름 충돌을 방지합니다.

03

testEx.cs — 클래스 정의

testEx.cs 
namespace PJ_2_1_Day04
{
    internal class testEx
    {
        // ① 배열 출력 메서드
        public void arrayEx()
        {
            string arr = "C#8";  // 문자열 == 문자의 배열
            Console.WriteLine(arr[0]);  // 'C'
            Console.WriteLine(arr[1]);  // '#'
            Console.WriteLine(arr[2]);  // '8'
        }

        // ② 사칙연산 계산기 메서드
        public void addEx()
        {
            Console.Write("첫 번째 숫자를 입력하세요: ");
            double num1 = Convert.ToDouble(Console.ReadLine());

            Console.Write("두 번째 숫자를 입력하세요: ");
            double num2 = Convert.ToDouble(Console.ReadLine());

            Console.Write("연산을 선택하세요 (+, -, *, /): ");
            string operation = Console.ReadLine();

            double result = 0;
            switch (operation)
            {
                case "+": result = num1 + num2; break;
                case "-": result = num1 - num2; break;
                case "*": result = num1 * num2; break;
                case "/":
                    if (num1 != 0 && num2 != 0)
                        result = num1 / num2;
                    else
                        Console.WriteLine("0으로 나눌 수 없습니다.");
                    break;
                default:
                    Console.WriteLine("잘못된 연산자입니다.");
                    break;
            }
            Console.WriteLine($"결과: {result}");
        }
    }
}
L3
internal class testEx — 클래스를 선언합니다. internal은 같은 프로젝트 안에서만 접근 가능하다는 의미입니다.
L6
public void arrayEx() — 메서드를 선언합니다. public은 외부에서 호출 가능, void는 반환값 없음을 뜻합니다.
L8
string arr = "C#8" — C#에서 문자열(string)은 문자(char)의 배열입니다. arr[0]은 'C', arr[1]은 '#', arr[2]는 '8'입니다.
L14
public void addEx() — 두 번째 메서드입니다. 사용자 입력을 받아 사칙연산을 수행합니다.
L28
switch (operation) — 입력된 연산자 문자열에 따라 분기합니다. default는 예상치 못한 입력을 처리합니다.
04

Program.cs — 객체 생성 및 메서드 호출

Program.cs 
namespace PJ_2_1_Day04
{
    internal class Program
    {
        static void Main()
        {
            // ① 객체 생성 (new 키워드)
            testEx print = new testEx();

            // ② 메서드 호출
            print.arrayEx();
            print.addEx();
        }
    }
}
L8
testEx print = new testEx(); — 설계도(클래스)를 바탕으로 객체를 실제로 만드는 코드입니다. print는 변수 이름(참조변수)입니다.
L11
print.arrayEx(); — 점(.) 연산자로 객체가 가진 메서드를 호출합니다. 객체이름.메서드이름() 형태입니다.
L12
print.addEx(); — 같은 객체로 두 번째 메서드도 호출합니다. 하나의 객체로 여러 메서드를 쓸 수 있습니다.
05

반드시 기억할 3단계 패턴

객체 사용의 3단계 
// STEP 1. 클래스 선언 (testEx.cs에서)
class 클래스이름 { /* 메서드 정의 */ }

// STEP 2. 객체 생성 (Program.cs에서)
클래스이름 변수이름 = new 클래스이름();

// STEP 3. 메서드 호출
변수이름.메서드이름();
✅

이 예제의 핵심: testEx라는 클래스에 arrayEx()와 addEx() 두 가지 기능을 담아두고, Program.cs에서 new testEx()로 객체를 하나 만든 뒤 필요한 메서드를 순서대로 호출합니다.

06

주요 요소 정리

요소 코드 예시 설명
클래스 선언 internal class testEx 기능의 설계도를 만듭니다
메서드 선언 public void arrayEx() 클래스 안에 기능을 정의합니다
객체 생성 new testEx() 설계도로 실제 객체를 만듭니다
메서드 호출 print.arrayEx() 객체의 기능을 실행합니다
문자열 인덱스 arr[0] 문자열의 각 문자에 접근합니다
형변환 Convert.ToDouble(...) 문자열 입력을 숫자로 변환합니다
⚠️

자주 하는 실수: static 메서드가 아닌 일반 메서드를 호출하려면 반드시 new로 객체를 먼저 만들어야 합니다. 객체 없이 바로 testEx.arrayEx()처럼 호출하면 오류가 납니다.
